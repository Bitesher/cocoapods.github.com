Installing CocoaPods

CocoaPods will be distributed as a binary by version 1.0. However for the time being you need to install it the ninja way!

Dependencies

  • Mac OS X 10.8
  • Ruby MRI 2.0.0 or 1.8.7 (ships with Mac OS X)
  • Xcode command line tools.

Installation

To install CocoaPods you can run:

$ [sudo] gem install cocoapods

If you want to have CocoaPods generate documentation for each library, then install the appledoc tool:

$ brew install appledoc

To enjoy performance benefits you can install a modern Ruby like 2.0.0 through a Ruby manager like RVM. If you don't use the system Ruby don't use sudo as it can create issues.

Update

To update CocoaPods you can run:

$ [sudo] gem update cocoapods

If you would like to try a pre-release version of CocoaPods you can run:

$ [sudo] gem install cocoapods-core --pre
$ [sudo] gem install cocoapods --pre

Notes

  • If you're using a fresh out of the box Mac using Xcode from the Mac App Store, you will need to install the Command Line Tools for Xcode first. The Command Line Tools can be found here or can be installed directly from Xcode:

    Xcode > Settings > Downloads > Components > Command Line Tools

  • If you use RVM, you might need to create a symbolic link to the LLVM GCC compiler:

    $ [sudo] ln -s /usr/bin/llvm-gcc-4.2 /usr/bin/gcc-4.2

  • CocoaPods is not compatible with MacRuby.

Troubleshooting

  • Missing command line tools

    ERROR: Failed to build gem native extension.
/System/Library/Frameworks/Ruby.framework/Versions/1.8/usr/bin/ruby extconf.rb
