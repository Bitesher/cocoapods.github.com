Sandbox

The sandbox provides support for the directory that CocoaPods uses for an installation. In this directory the Pods projects, the support files and the sources of the Pods are stored.

CocoaPods assumes to have control of the sandbox.

Once completed the sandbox will have the following file structure:

Pods
|
+-- User
| +-- [Target Name]-configuration.h
| +-- Specs
| +-- Scripts
|
+-- Generated
  +-- Headers
  |   +-- Private
  |   |   +-- [Pod Name]
  |   +-- Public
  |       +-- [Pod Name]
  |
  +-- Sources
  |   +-- [Pod Name]
  |
  +-- Specs
  |   +-- External Sources
  |   +-- Normal Sources
  |
  +-- Target Support Files
  |   +-- [Target Name]
  |       +-- Pods-acknowledgements.markdown
  |       +-- Pods-acknowledgements.plist
  |       +-- Pods-dummy.m
  |       +-- Pods-prefix.pch
  |       +-- Pods.xcconfig
  |
  +-- Manifest.lock
  |
  +-- Pods.xcodeproj

See #833

229 def store_podspec(name, podspec, external_source = false)
230   output_path = specifications_dir(external_source) + "#{name}.podspec"
231   output_path.dirname.mkpath
232   if podspec.is_a?(String)
233     output_path.open('w') { |f| f.puts(podspec) }
234   else
235     unless podspec.exist?
236       raise Informative, "No podspec found for `#{name}` in #{podspec}"
237     end
238     FileUtils.copy(podspec, output_path)
239   end
240   spec = Specification.from_file(output_path)
241   unless spec.name == name
242     raise Informative, "The name of the given podspec `#{spec.name}` doesn't match the expected one `#{name}`"
243   end
244 end
