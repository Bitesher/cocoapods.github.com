Sandbox

The sandbox provides support for the directory that CocoaPods uses for an installation. In this directory the Pods projects, the support files and the sources of the Pods are stored.

CocoaPods assumes to have control of the sandbox.

Once completed the sandbox will have the following file structure:

Pods
|
+-- User
| +-- [Target Name]-configuration.h
| +-- Specs
| +-- Scripts
|
+-- Generated
  +-- Headers
  |   +-- Private
  |   |   +-- [Pod Name]
  |   +-- Public
  |       +-- [Pod Name]
  |
  +-- Sources
  |   +-- [Pod Name]
  |
  +-- Specs
  |   +-- External Sources
  |   +-- Normal Sources
  |
  +-- Target Support Files
  |   +-- [Target Name]
  |       +-- Pods-acknowledgements.markdown
  |       +-- Pods-acknowledgements.plist
  |       +-- Pods-dummy.m
  |       +-- Pods-prefix.pch
  |       +-- Pods.xcconfig
  |
  +-- Manifest.lock
  |
  +-- Pods.xcodeproj

See #833

224 def store_podspec(name, podspec, external_source = false)
225   output_path = specifications_dir(external_source) + "#{name}.podspec"
226   output_path.dirname.mkpath
227   if podspec.is_a?(String)
228     output_path.open('w') { |f| f.puts(podspec) }
229   else
230     unless podspec.exist?
231       raise Informative, "No podspec found for `#{name}` in #{podspec}"
232     end
233     FileUtils.copy(podspec, output_path)
234   end
235   spec = Specification.from_file(output_path)
236   unless spec.name == name
237     raise Informative, "The name of the given podspec `#{spec.name}` doesn't match the expected one `#{name}`"
238   end
239 end
