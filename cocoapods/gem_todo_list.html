CocoaPods Todos

Pod::Command#initialize

If a command is run inside another one some settings which where true might return false.
We should probably not even load colored unless needed.

Pod::Command::Push#add_specs_to_repo

Raise if the source is not under git source control.

Pod::Command::Spec#default_data_for_template

It would be nice to have a template class that accepts options and uses the default ones if not provided.
The template is outdated.

Pod::CoreUI::puts

Enable in CocoaPods 0.17.0 release

Pod::CoreUI::warn

Enable in CocoaPods 0.17.0 release

Pod::Executable::execute_command

Find a way to display the live output of the commands.

Pod::Generator::Documentation#generate

Passing the files explicitly clutters output and chokes on very long list (AWSiOSSDK). It is possible to just pass the dir of the pod, however this would include other files like demo projects.

Pod::Generator::Documentation#company

Set to CocoaPods?

Pod::Generator::PrefixHeader#generate

Subspecs can specify prefix header information too.

Pod::Installer#clean_sandbox

[#247] Clean the headers of only the pods to install.

Pod::Installer#prepare_pods_project

Clean and modify the project if it exists.

Pod::Installer#write_lockfiles

Pass the checkout options to the Lockfile.

Pod::Installer#integrate_user_project

[#397] The libraries should be cleaned and the re-added on every installation. Maybe a clean_user_project phase should be added. In any case it appears to be a good idea store target definition information in the lockfile.

Pod::Installer::Analyzer#generate_podfile_state

[CocoaPods > 0.18] If there isn't a Lockfile all the Pods should be marked as added.

Pod::Installer::Analyzer#compute_platform_for_target_definition

Is assigning the platform to the target definition the best way to go?

Pod::Installer::Analyzer::SandboxAnalyzer#lockfile

Remove for CP 0.18.

Pod::Installer::TargetInstaller#add_target

Add integration test for build configurations and don't add the build configurations to the project if they are not needed.

Pod::Installer::UserProjectIntegrator#installation_root

This is only used to compute the workspace path in case that it should be inferred by the project. If the workspace should be in the same dir of the project, this could be removed.

Pod::Installer::UserProjectIntegrator#initialize

Too many initialization arguments

Pod::Installer::UserProjectIntegrator::TargetIntegrator#add_xcconfig_base_configuration

If the xcconfig is already set don't override it and inform the user.

Pod::Sandbox#specifications_dir

Migrate old installations and store the for all the pods. Two folders should be created External Sources and Podspecs.

Pod::Sandbox#store_podspec

Store all the specifications (including those not originating from external sources) so users can check them.

Pod::Sandbox::FileAccessor#glob_for_attribute

Move to the cocoapods-core so it appears in the docs?

Pod::Sandbox::FileAccessor#expanded_paths

Implement case insensitive search

Pod::UserInterface::section

Refactor to title (for always visible titles like search) and sections (titles that represent collapsible sections).

Pod::UserInterface::message

Clean interface.
