CocoaPods Todos

Pod::Command#initialize

If a command is run inside another one some settings which where true might return false.
We should probably not even load colored unless needed.

Pod::Command::Push#add_specs_to_repo

Raise if the source is not under git source control.

Pod::CoreUI::warn

Enable in CocoaPods 0.17.0 release

Pod::Executable::execute_command

Find a way to display the live output of the commands.

Pod::ExternalSources::AbstractExternalSource#store_podspec

This could be done by the sandbox.
Store all the specifications (including those not originating from external sources) so users can check them.
The check for the podspec string is a bit primitive.

Pod::Generator::Documentation#generate

Passing the files explicitly clutters output and chokes on very long list (AWSiOSSDK). It is possible to just pass the dir of the pod, however this would include other files like demo projects.

Pod::Generator::PrefixHeader#generate

Subspecs can specify prefix header information too.

Pod::Installer#generate_local_pods

[#535] LocalPods should resolve the path of the specifications passing the library as arguments.
Why the local pods are generated by the sandbox? I guess because some where pre-downloaded? However the sandbox should just store the name of those Pods.

Pod::Installer#generate_names_of_pods_to_install

[#534] Detect if the folder of a Pod is empty (even if it exits).
There could be issues with the current implementation regarding external specs.

Pod::Installer#prepare_sandbox

[#247] Clean the headers of only the pods to install.
[#534] Clean all the Pods folder that are not unchanged
Use the local pod implode.
Clean the podspecs of all the pods that aren't unchanged so the resolution process doesn't get confused by them.

Pod::Installer#prepare_pods_project

Clean and modify the project if it exists.

Pod::Installer#add_file_references_to_pods_project

Clean the groups of the deleted Pods and add only the Pods that should be installed.

Pod::Installer#run_pre_install_hooks

Run the hooks only for the installed pods.
Print a message with the names of the specs.

Pod::Installer#run_post_install_hooks

Run the hooks only for the installed pods.
Print a message with the names of the specs.

Pod::Installer#integrate_user_project

[#397] The libraries should be cleaned and the re-added on every installation. Maybe a clean_user_project phase should be added. In any case it appears to be a good idea store target definition information in the lockfile.
[#588] The resources should be added through a build phase instead of using a script.

Pod::Installer#download_pod

Store the source of non specific downloads in the lockfile.

Pod::Installer::Analyzer#generate_podfile_state

[CocoaPods > 0.18] If there isn't a Lockfile all the Pods should be marked as added.

Pod::Installer::Analyzer#generate_sandbox_state

[CocoaPods > 0.18] Version 0.17 falls back to the lockfile of the Podfile for the sandbox manifest to prevent the full re-installation for upgrading users (this was the old behaviour pre sandbox manifest) of all the pods. Drop in 0.18.

Pod::Installer::Analyzer#compute_platform_for_target_definition

Is assigning the platform to the target definition the best way to go?

Pod::Installer::TargetInstaller#prefix_header_filename

This has to be removed, but this means the specs have to be updated if they need a reference to the prefix header.

Pod::Installer::TargetInstaller#add_target

Add integration test for build configurations and don't add the build configurations to the project if they are not needed.

Pod::Installer::UserProjectIntegrator#project_root

This is only used to compute the workspace path in case that it should be inferred by the project. If the workspace should be in the same dir of the project, this could be removed.

Pod::Installer::UserProjectIntegrator#initialize

Too many initialization arguments

Pod::Installer::UserProjectIntegrator::TargetIntegrator#add_xcconfig_base_configuration

If the xcconfig is already set don't override it and inform the user.

Pod::LocalPod#initialize

The local pod should be initialized with all the activated specifications passed as an array, in order to be able to cache the computed values. In other words, it should be immutable.

Pod::LocalPod#clean!

[#529] Podspecs should not be preserved anymore to prevent user confusion. Currently we are copying the ones form external sources in Local Podspecs and this feature is not needed anymore. I think that copying all the used podspecs would be helpful for debugging.

Pod::LocalPod#header_mappings

This is being overridden in the RestKit 0.9.4 spec and that override should be fixed.

Pod::LocalPod#expanded_paths

Implement case insensitive search

Pod::UserInterface::section

Refactor to title (for always visible titles like search) and sections (titles that represent collapsible sections).

Pod::UserInterface::message

Clean interface.
