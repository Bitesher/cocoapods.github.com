DSL

The Podfile is a specification that describes the dependencies of the targets of one or more Xcode projects. The Podfile always creates an implicit target, named default, which links to the first target of the user project.

A podfile can be very simple:

pod 'AFNetworking', '~> 1.0'

An example of a more complex podfile can be:

platform :ios, '6.0'
inhibit_all_warnings!

xcodeproj `MyProject`

pod 'ObjectiveSugar', '~> 0.5'

target :test do
  pod 'OCMock', '~> 2.0.1'
end

post_install do |installer|
  installer.project.targets.each do |target|
    puts "#target.name"
  end
end

166 def podspec(options = nil)
167   if options && path = options[:path]
168     path = File.extname(path) == '.podspec' ? path : "#{path}.podspec"
169     file = Pathname.new(File.expand_path(path))
170   elsif options && name = options[:name]
171     name = File.extname(name) == '.podspec' ? name : "#{name}.podspec"
172     file = defined_in_file.dirname + name
173   else
174     file = Pathname.glob(defined_in_file.dirname + '*.podspec').first
175   end
176 
177   spec = Specification.from_file(file)
178   spec.activate_platform(@target_definition.platform)
179   all_specs = spec.recursive_subspecs.push(spec)
180   deps = all_specs.map {|specification| specification.external_dependencies }
181   deps = deps.flatten.uniq
182   @target_definition.target_dependencies.concat(deps)
