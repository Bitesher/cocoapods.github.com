CocoaPods-Core Todos

Pod::Dependency#requirement

The specific version is stripped from head information because because its string representation would not parse. It would be better to add something like Version#display_string.

Pod::Dependency#to_root_dependency

This should not use dup. The name property should be an attr_reader.

Pod::Dependency#external_source_description

Improve the description for Mercurial and Subversion.

Pod::Lockfile#detect_changes_with_podfile

Why do we look for compatibility instead of just comparing if the two dependencies are equal?

Pod::Lockfile::generate_pods_data

Specifications should be stored per platform, otherwise they list dependencies which actually might not be used.

Pod::Lockfile::generate_external_sources_data

The downloader should generate an external source hash that should be store for dependencies in head mode and for those with external source.

Pod::Podfile::TargetDefinition#store_pod

This needs urgently a rename.

Pod::Podfile::TargetDefinition#store_podspec

This needs urgently a rename.

Pod::Source::Aggregate#search_by_name

Clients should raise not this method.

Pod::Specification#==

Not sure if comparing only the name and the version is the way to go. This is used by the installer to group specifications by root spec.

Pod::Specification::DSL::RootAttributesAccessors#version

The version is memoized because the Resolvers sets the head state on it. This information should be stored in the specification instance and the lockfile should have a more robust handling of head versions (like a dedicated section).

Pod::Specification::Linter#validate_file_patterns

Check the attributes hash directly.

Pod::Specification::Linter#check_tmp_arc_not_nil

Remove in 0.18 and switch the default to true.

Pod::Specification::Set#required_by

This should simply return a boolean. Is CocoaPods that should raise.

Pod::Specification::Set#required_version

This should simply return nil. CocoaPods should raise instead.

Pod::Version#initialize

The from part of the regular expression should be remove in CocoaPods 1.0.0.

Pod::Version#to_s

Adding the head information to the string representation creates issues (see Dependency#requirement).

Pod::YAMLConverter::sorted_array

This stuff is here only because the Lockfile intermixes strings and hashes for the PODS key. The Lockfile should be more consistent.
